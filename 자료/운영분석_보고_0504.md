## 1. AI 검색 처리 구조

### 1-1. 서비스 개요
- 고용노동부 다우리포털에 임베드된 AI 직원 검색 위젯
- LDAP 인사 데이터 14,731명 기반, 이름·부서·직급·업무 등 자연어 검색 지원
- 지능정보화기획팀 자체 개발·운영 (외부 솔루션 미사용)

### 1-2. 검색 처리 흐름

```
사용자 입력 → 토큰 분류 → 단축회로(76%) / AI 자연어(23%) → SQL 생성 → 결과 반환
```

**① 토큰 분류 (Token Classification)**
- 입력 쿼리를 단어 단위로 분석, 10가지 유형으로 자동 분류
- 분류 유형: 기관명·부서명·직급·직명·상태(휴직/파견)·업무 키워드·지역명·이름 등
- 사전 로드된 조직·직급·업무 사전과 즉시 매칭

**② 단축회로 처리 — 전체의 76%, 평균 0.01초**
- 토큰 분류 결과만으로 SQL 조건 즉시 생성, LLM 호출 불필요
- 대부분의 검색을 AI 없이 0.01초 이내 처리 가능
- 예: "서울청 근로감독관" → `org LIKE '%서울%' AND duty_nm = '근로감독관'`

**③ AI 자연어 처리 — 전체의 23%, 평균 1.9초**
- 단축회로로 처리 불가한 복합 조건은 내부망 전용 LLM에 전달
- 자연어 → JSON 필터 → SQL 2단계 변환으로 복합 검색 조건 자동 생성
- 예: "부산청 근로감독관 중 5급 이상 산업안전 담당자" → 4개 조건 복합 SQL 자동 생성
- 내부망 폐쇄 환경에서 운용, 외부 데이터 유출 원천 차단

**④ 결과 후처리**
- 직급순 → 기관유형순(본부→지방청→지청) → 사번순 자동 정렬
- LDAP 직원 사진 연동, IP전화 바로걸기(Click-to-Call), 메일 연동

### 1-3. 기존 LDAP 검색 대비 도입 효과

| 항목 | 기존 (LDAP 직접 연결) | 현재 (AI 검색 위젯) |
|------|----------------------|---------------------|
| 검색 방식 | LDAP 서버 풀 연결·로드 | 로컬 DB 사전 적재 + 토큰 분류 |
| 응답 속도 | 3~5초 이상 | **0.01초** (단순) / 1.9초 (자연어) |
| 검색 기능 | 이름·부서 단순 매칭만 가능 | 자연어·복합조건·업무키워드 검색 지원 |
| 접근 방식 | 별도 페이지 이동 필요 | 포털 모든 화면에서 즉시 검색 (Ctrl+I) |
| 네트워크 부하 | 매 검색마다 LDAP 서버 통신 | 로컬 DB 조회, LDAP 서버 부하 없음 |

- **응답 속도 300배 이상 개선**: 기존 3~5초 → 단순 검색 0.01초 이내 즉시 응답
- **LDAP 서버 부하 해소**: 매 검색마다 발생하던 LDAP 서버 직접 조회를 제거, 사전 동기화된 로컬 DB 활용으로 네트워크 트래픽 대폭 감소
- **검색 기능 고도화**: 단순 이름·부서 매칭에서 자연어 복합 조건 검색, 업무 키워드 검색, 직명·상태 검색으로 확장
- **접근성 향상**: 별도 페이지 이동 없이 위젯·단축키(Ctrl+I)로 포털 어디서든 즉시 검색 가능

### 1-4. 지속 개선 체계
- 검색 로그 전수 분석 → 0건 검색 원인 파악 → 토큰 분류 규칙·LLM 프롬프트 지속 튜닝
- 검색 실패율: 오픈 초기 13.9% → 현재 0% (1,000회 자동화 테스트 검증)
- 사용자 피드백 직접 수집·분석, 검색 로직 개선에 반영
