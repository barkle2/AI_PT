# 박태웅 의장의 공공 AX 관점

박태웅 국가AI전략위원회 공공AX분과장은 2026년 4월 29일 '사회보장 AX 미래전략 심포지엄' 기조강연에서 공공 AX의 현실을 날카롭게 진단하고, AI 전환의 전제 조건과 방향성을 제시했다.

---

## 1. 핵심 관점: "데이터 없는 AX는 시늉이다"

박태웅 의장의 공공 AX 관점을 한 문장으로 압축하면 이것이다.

> "AX를 추진하기에 앞서 데이터 현황을 점검하는 일이 첫째다. 필요 데이터가 무엇인지 모르는 채로 AX를 할 수는 없다."

그는 공공부문에서 AI 관련 사업이 쏟아지고 있지만, 실제 AX가 이뤄질 기반이 마련돼 있지 않다고 비판한다. 데이터가 없는 상황에서 AI를 도입하면 **"결과를 낸 것처럼 시늉만 할 뿐"**이라는 것이 그의 핵심 진단이다.

---

## 2. 현재 공공부문의 문제 진단

### 로그가 없는 정부
- 정부 사이트 대부분이 **로그(Log)를 기록하지 않는다.**
- 로그가 없으면 어떤 메뉴를 자주 쓰는지, 로딩 속도가 얼마나 걸리는지조차 파악 불가
- AI는 데이터를 학습해 패턴을 파악하는데, 정부 사이트는 **사실상 데이터가 없는 상태**
- 이 기반 위에서 AI는 아무 일도 할 수 없다

### 소프트웨어화가 안 된 정부
- 데이터를 연결할 수 있는가? 기계가 읽을 수 있는가?
- FAIR한가? 표준 API가 있는가? 표준 MCP가 있는가?
- 종이 문서 5장 수준에 머물러 있는 공공 데이터 → 시계열 분석이 불가능한 구조

### 부처 간 칸막이
- 복지 수급 이력, 공공요금 체납, 건강보험 진료 명세 등 데이터가 부처별로 단절
- 통합 없이는 국민 한 명을 위한 AI 기반 선제적 복지 불가능

---

## 3. AX의 선결 조건 (박태웅 의장이 강조하는 순서)

### 1단계: 데이터 확보 — 소프트웨어화
- 모든 문서·자료를 **기계가 읽을 수 있는 데이터로 재구성**
- 로그 수집부터 시작 → 사용 패턴 파악 → AI 학습 기반 마련
- "데이터를 준비하고 AI 도입을 고민하는 절차가 선행되어야 한다"

### 2단계: FAIR 원칙 적용
- **F**indable(검색 가능): 데이터를 찾을 수 있어야 함
- **A**ccessible(접근 가능): 인증·인가를 거쳐 접근 가능해야 함
- **I**nteroperable(상호운용): 다른 데이터와 연결·통합 가능해야 함
- **R**eusable(재사용 가능): 데이터의 출처와 이력(provenance)이 명확해야 함

### 3단계: 인프라 구축 — Git·MCP·OpenAPI
- **Git**: 법령 전체(6,907건)를 버전관리 — 버전 비교, 법령 개정 이력 추적 가능
- **MCP(Model Context Protocol)**: AI가 메뉴얼·도구와 함께 작동하는 프로토콜
  - AI가 직접 사이트에 들어가 검색·판례 확인 → 관련 규정을 찾아 답
- **Open API/OpenMCP**: 국민이 직접 공공서비스를 개발할 수 있는 환경

### 4단계: 거버넌스 확립
- "공공AX 거버넌스 확립도 시급히 해결해야 할 핵심 선결 과제"
- 모든 데이터는 **표준화·통합·현재화** 되어야 함
- 부처 간 칸막이 해소, API 형태의 데이터 공유 체계 필요

## 4. 박태웅 의장이 그리는 공공 AX의 지향점

### AI를 활용한 최고의 대국민 서비스 8원칙
1. 정부의 모든 서비스는 하나의 아이디로 접근 가능해야 한다
2. 로그시스템을 보유하고, 고장 시 신속 복구 체계를 갖춰야 한다
3. 개인의 상황에 맞는 맞춤형 판단을 반드시 제공해야 한다
4. 자격이 있으면 신청 없이도 자동으로 서비스를 제공해야 한다
5. 금융·통신 등 주요 민간서비스와 연계해야 한다
6. 자격 여부와 관계없이 누구나 필요한 서비스를 받을 수 있어야 한다
7. 필요한 데이터를 표준화하고 통합해야 한다
8. Open API·OpenMCP를 공개해 국민이 직접 서비스를 개발하고 참여할 수 있어야 한다

### AI Native 정부의 모습
- **오늘의 일이 내일의 Data가 되는 정부** — 업무 수행이 곧 데이터 축적
- 모든 업무가 클라우드 기반으로 이뤄지는 정부
- 데이터가 스스로 연결되는 정부
- 자격이 있으면 신청이 필요 없는 정부 (선제적 서비스)
- 일을 할수록 스마트해지는 정부

## 5. AI 시대 변화에 대한 시각

- AI 발전 = **산업혁명에 비견되는 변화** — 증기기관이 육체노동을 대체했듯, AI는 인간의 정신노동을 대체
- AI 시대엔 **코딩을 모르는 현업도 직접 프로그램 개발 가능** (korean-law-mcp 사례)
- 정부 SI 발주 → 대기업 → 중소기업으로 이어지는 구조의 **근본적 변화** 예고
- 한국의 AI 역량은 세계 최고 수준 — "정말 잘해야 한다"

## 6. 고용노동부 AX 전략에 대한 시사점

| 박태웅 의장 강조 사항 | 고용노동부 적용 포인트 |
|---|---|
| 로그 수집부터 시작 | 고용서비스 포털(워크넷 등) 로그 현황 점검 |
| 소프트웨어화·FAIR 데이터 | 근로감독·산재 데이터의 기계가독성 확보 |
| 부처 간 칸막이 해소 | 복지부·행안부·국세청과 고용 데이터 연계 |
| 자격 있으면 자동 서비스 | 실업급여·고용장려금 선제적 안내 |
| Open API 공개 | 고용 데이터 외부 활용 허용 |
| AX 거버넌스 확립 | 고용노동부 AX 추진 체계·데이터 전략 수립 선행 |
