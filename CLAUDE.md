## 작업 시작 규칙

**중요: 사용자가 명시적으로 "작업 시작" 또는 이에 준하는 명령을 내리기 전까지 어떠한 파일 수정, 생성, 삭제도 수행하지 않는다.**

### 허용되는 행동 (작업 시작 전)
- 질문에 답변하기
- 파일 읽기 및 코드 분석
- 계획 제안 및 접근법 설명
- 사용자의 요청 내용 확인 및 명확화 질문

### 금지되는 행동 (작업 시작 전)
- 파일 생성, 수정, 삭제
- 코드 실행 (빌드, 테스트 등)
- git 명령 실행
- 외부 서비스 호출

### 작업 시작 트리거 예시
사용자가 아래와 같이 말할 때만 실제 작업을 시작한다:
- "작업 시작"
- "시작해줘"
- "진행해줘"
- "해줘" / "적용해줘"
- "OK" / "Go ahead"

### 기본 동작
요청을 받으면:
1. 요청 내용을 이해하고 분석한다
2. 수행할 작업 계획을 설명한다
3. **사용자의 명시적 승인을 기다린다**
4. 승인 후 작업을 실행한다

## 대규모 작업 처리 규칙 (Task Chunking)
* 작업시간이 오래 걸리거나(10분 이상), 토큰 소모가 많을 것으로 예상되는(10000 토큰 이상) 다량의 파일 수정, 긴 문서 작성 등의 작업은 **반드시 작은 단위로 쪼개서(Step-by-step) 진행**할 것.
* 한 번에 모든 것을 처리하려 하지 말고, 먼저 **수행할 작업의 단위(Plan)를 목차 형태로 제시**한 뒤, 한 단계씩 실행하고 다음 단계로 넘어갈 것.
